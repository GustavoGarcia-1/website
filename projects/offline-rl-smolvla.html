Offline RL + smolVLA

In Progress

Overview

This project combines offline reinforcement learning with vision-language-action (VLA) pretraining to improve simulated robotic manipulation. The core idea: take a smolVLA base model pretrained on vision-language-action data, then apply Physical Intelligence's pi0.6 RECAP offline RL framework to fine-tune it for specific manipulation tasks in MuJoCo — without any additional online data collection.

The goal is to show that offline RL fine-tuning on top of a VLA backbone can improve sample efficiency and task performance compared to either approach alone. By leveraging the generalization of VLA pretraining and the policy improvement of offline RL, the system should learn better manipulation policies from fixed datasets.

Architecture

The architecture uses smolVLA as the base vision-language-action model, providing a pretrained backbone that maps visual observations and language instructions to action predictions. The pi0.6 RECAP framework is layered on top, applying offline RL fine-tuning to improve the policy using previously collected trajectory data without requiring new environment interactions.

# Architecture overview (placeholder)
# smolVLA base model → pi0.6 RECAP offline RL layer

class OfflineRLVLAPolicy:
    def __init__(self, vla_checkpoint: str, recap_config: dict):
        self.vla_backbone = load_smolvla(vla_checkpoint)
        self.recap_head = RECAPHead(recap_config)

    def forward(self, obs, language_instruction):
        vla_features = self.vla_backbone.encode(obs, language_instruction)
        actions = self.recap_head.predict(vla_features)
        return actions

Simulation Environment

The manipulation environment is built in MuJoCo, featuring a tabletop setup with a robot arm and objects to manipulate. The environment provides RGB image observations from one or more camera views, proprioceptive state, and accepts continuous joint-space or end-effector actions.

envs/manipulation_env.py (placeholder)
# MuJoCo manipulation environment (placeholder)

class ManipulationEnv:
    def __init__(self, task: str, camera_names: list[str]):
        self.model = mujoco.MjModel.from_xml_path(get_task_xml(task))
        self.data = mujoco.MjData(self.model)
        self.cameras = camera_names

    def get_obs(self) -> dict:
        images = {cam: self.render(cam) for cam in self.cameras}
        proprio = self.get_proprioception()
        return {"images": images, "proprio": proprio}

    def step(self, action):
        self.data.ctrl[:] = action
        mujoco.mj_step(self.model, self.data)
        return self.get_obs(), self.compute_reward(), self.check_done()
