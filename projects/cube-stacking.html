Cube Stacking Robot

Overview

This project tests multiple vision language action models on a cube stacking task using the so101 robotic arms as part of Triton Droids. I trained and compared three models, smolVLA, ACT (Action Chunking with Transformers), and π0.5, on a shared teleoperation dataset to understand their relative strengths and failure modes for precise pick and place manipulation.

After comparing all three, π0.5 performed the best,smolVLA had trouble with the complexity of the task as I believe it exceeded what the smaller model could handle. ACT showed promise and we felt it would have performed well with more tuning, but π0.5 performed the best

Training was distributed across H100 GPUs via VESSL AI, Google Colab, and the San Diego Supercomputer Center. All checkpoints were uploaded to HuggingFace for reproducibility.

Data Collection

Hours of teleoperation data were collected on the so101 arms, with operators performing cube stacking sequences of varying difficulty. Each episode was recorded with synchronized multi view cameras (wrist and eagle eye) to provide rich visual context for the policy. The setup entirely based on the LeRobot recording framework.

lerobot-record \
    --robot.type=so101_follower \
    --robot.port=/dev/tty.usbmodem5A7A0570191 \
    --robot.id=triton_follower \
    --robot.cameras="{ front: {type: opencv, index_or_path: 0, width: 640, height: 480, fps: 30, fourcc: MJPG},
      side: {type: opencv, index_or_path: 1, width: 640, height: 480, fps: 30, fourcc: MJPG}}" \
    --teleop.type=so101_leader \
    --teleop.port=/dev/tty.usbmodem5AB01824781 \
    --teleop.id=triton_leader \
    --display_data=true \
    --dataset.repo_id=Gueso/cube_stacking \
    --dataset.single_task="Pick up the cube and stack it on top of the other cube." \
    --dataset.episode_time_s=20 \
    --dataset.reset_time_s=15 \
    --resume=true

Model Comparison

Each model was trained on the same dataset using the LeRobot training pipeline. We used the existing LeRobot training code without modification, just swapping out the policy type and dataset config for each run.

SmolVLA is a lightweight VLA model designed for reproducibility. It uses a compact vision encoder and smaller language backbone, making it faster to train and run inference on. However for cube stacking, the model struggled to produce consistent grasping behavior. We believe the task required more representation capacity than the smaller architecture could provide.

ACT (Action Chunking with Transformers) predicts chunks of future actions using a CVAE encoder decoder architecture. ACT's action chunking approach helps with temporal consistency, reducing compounding error in sequential prediction. We had some trouble getting it running but felt it would perform well given more data and less cube position variation.

π0.5 is Physical Intelligence's flow matching policy that generates continuous action trajectories. It performed the best of the three models on this task, producing smooth and precise motions with the only successful completitons out of the three.

python lerobot_train.py \
    --dataset.repo_id=Gueso/cube_stacking \
    --policy.type=pi05 \
    --policy.pretrained_path=lerobot/pi05_base \
    --policy.normalization_mapping='{"ACTION": "MEAN_STD", "STATE": "MEAN_STD", "VISUAL": "IDENTITY"}' \
    --policy.compile_model=true \
    --policy.gradient_checkpointing=true \
    --policy.dtype=bfloat16 \
    --policy.device=cuda \
    --policy.repo_id=Gueso/pi05_cubestacking \
    --output_dir=./outputs/pi05_cubestacking \
    --batch_size=16 \
    --steps=50000 \
    --save_freq=2000

Results and Takeaways

π0.5 was the best model for this task, completing the full stacking sequence. However, it did not consistently complete the task it had a success rate of ~10%, Smolvla 0%, ACT 0%. SmolVLA struggled with grasping accuracy and jerky motions. ACT and Smolvla failed because of the cube position variation in our data.

Larger models like π0.5 can handle the complexity of multi step pick and place, and are able to generalize well with different cube positions. While smaller models may need significantly more data or a simpler task definition to succeed.

All checkpoints were uploaded to HuggingFace, and training was tracked via Weights and Biases across H100 GPUs on VESSL AI, Google Colab, and SDSC.
