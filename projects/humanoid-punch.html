Humanoid Punch (URKL)

Overview

This project is part of URKL (Intense Humanoid Robot Combat) at Triton Droids. The goal: train an EngineAI T800 humanoid robot to throw a punch using reinforcement learning and motion capture imitation. The pipeline converts real boxing mocap data into joint trajectories, then uses PPO to train a policy that tracks the full-body motion — including maintaining balance while delivering a strike.

The system is built on MJLab, a MuJoCo-based RL framework with RSL-RL as the training backend. The T800 model has 30 DOFs tracked across 14 body segments, with self-collision detection and adaptive curriculum sampling to handle the difficulty of violent, asymmetric motions like punches.

Mocap Data & Motion Loading

Boxing motion capture data is converted from BVH format into NPZ files containing joint positions, joint velocities, body positions, orientations, and linear/angular velocities for every tracked body segment at each timestep. The MotionLoader class indexes into this data at runtime, mapping mocap body segments to the T800's kinematic tree.

mjlab/src/mjlab/tasks/tracking/mdp/commands.py
class MotionLoader:
  def __init__(
    self, motion_file: str, body_indexes: torch.Tensor, device: str = "cpu"
  ) -> None:
    data = np.load(motion_file)
    self.joint_pos = torch.tensor(data["joint_pos"], dtype=torch.float32, device=device)
    self.joint_vel = torch.tensor(data["joint_vel"], dtype=torch.float32, device=device)
    self._body_pos_w = torch.tensor(
      data["body_pos_w"], dtype=torch.float32, device=device
    )
    self._body_quat_w = torch.tensor(
      data["body_quat_w"], dtype=torch.float32, device=device
    )
    self._body_lin_vel_w = torch.tensor(
      data["body_lin_vel_w"], dtype=torch.float32, device=device
    )
    self._body_ang_vel_w = torch.tensor(
      data["body_ang_vel_w"], dtype=torch.float32, device=device
    )
    self._body_indexes = body_indexes
    self.body_pos_w = self._body_pos_w[:, self._body_indexes]
    self.body_quat_w = self._body_quat_w[:, self._body_indexes]
    self.body_lin_vel_w = self._body_lin_vel_w[:, self._body_indexes]
    self.body_ang_vel_w = self._body_ang_vel_w[:, self._body_indexes]
    self.time_step_total = self.joint_pos.shape[0]

Reward Shaping

The reward function uses multiple exponential-error components to track the mocap reference across the full body. Each term computes a Gaussian kernel over the squared error between the reference and robot state — providing smooth gradients everywhere, even far from the target. Separate terms handle anchor body position/orientation, relative body positions, and linear/angular velocities. A self-collision penalty discourages the robot from intersecting its own limbs during the punch.

mjlab/src/mjlab/tasks/tracking/mdp/rewards.py
def motion_global_anchor_position_error_exp(
  env: ManagerBasedRlEnv, command_name: str, std: float
) -> torch.Tensor:
  command = cast(MotionCommand, env.command_manager.get_term(command_name))
  error = torch.sum(
    torch.square(command.anchor_pos_w - command.robot_anchor_pos_w), dim=-1
  )
  return torch.exp(-error / std**2)


def motion_relative_body_position_error_exp(
  env: ManagerBasedRlEnv,
  command_name: str,
  std: float,
  body_names: tuple[str, ...] | None = None,
) -> torch.Tensor:
  command = cast(MotionCommand, env.command_manager.get_term(command_name))
  body_indexes = _get_body_indexes(command, body_names)
  error = torch.sum(
    torch.square(
      command.body_pos_relative_w[:, body_indexes]
      - command.robot_body_pos_w[:, body_indexes]
    ),
    dim=-1,
  )
  return torch.exp(-error.mean(-1) / std**2)


def self_collision_cost(
  env: ManagerBasedRlEnv,
  sensor_name: str,
  force_threshold: float = 10.0,
) -> torch.Tensor:
  """Penalize self-collisions.

  When the sensor provides force history (from history_length > 0),
  counts substeps where any contact force exceeds force_threshold.
  """
  sensor: ContactSensor = env.scene[sensor_name]
  data = sensor.data
  if data.force_history is not None:
    force_mag = torch.norm(data.force_history, dim=-1)  # [B, N, H]
    hit = (force_mag > force_threshold).any(dim=1)      # [B, H]
    return hit.sum(dim=-1).float()                     # [B]
  return data.found.squeeze(-1)

PPO Configuration & Training

Training uses PPO via RSL-RL with a 3-layer MLP (512 → 256 → 128) for both actor and critic. The policy uses a Gaussian action distribution with adaptive KL-based learning rate scheduling. The agent trains for up to 30,000 iterations with 24 environment steps per rollout, with checkpoints saved every 500 iterations. Training is tracked via Weights & Biases.

mjlab/src/mjlab/tasks/tracking/config/t800/rl_cfg.py

T800 Environment & Body Tracking

The environment tracks 14 body segments on the T800 humanoid — from the base link through hips, knees, ankles, shoulders, and elbows on both sides, anchored at the waist. Self-collision detection uses a contact sensor with 4-step force history to catch collisions during fast motions. Domain randomization includes base COM perturbation and joint position noise.

mjlab/src/mjlab/tasks/tracking/config/t800/env_cfgs.py

Adaptive Curriculum Sampling

One of the harder parts of training a punch motion is that the policy tends to fail at specific points in the trajectory — usually during the fast arm extension. The adaptive sampling system tracks which timestep bins the policy fails at most often, then oversamples those regions during training. This is implemented as a convolved failure histogram with exponential smoothing, so the curriculum shifts focus to the hardest parts of the motion automatically.

mjlab/src/mjlab/tasks/tracking/mdp/commands.py
def _adaptive_sampling(self, env_ids: torch.Tensor):
    episode_failed = self._env.termination_manager.terminated[env_ids]
    if torch.any(episode_failed):
      current_bin_index = torch.clamp(
        (self.time_steps * self.bin_count) // max(self.motion.time_step_total, 1),
        0, self.bin_count - 1,
      )
      fail_bins = current_bin_index[env_ids][episode_failed]
      self._current_bin_failed[:] = torch.bincount(fail_bins, minlength=self.bin_count)

    # Smooth failure histogram with convolution kernel
    sampling_probabilities = (
      self.bin_failed_count + self.cfg.adaptive_uniform_ratio / float(self.bin_count)
    )
    sampling_probabilities = torch.nn.functional.pad(
      sampling_probabilities.unsqueeze(0).unsqueeze(0),
      (0, self.cfg.adaptive_kernel_size - 1),
      mode="replicate",
    )
    sampling_probabilities = torch.nn.functional.conv1d(
      sampling_probabilities, self.kernel.view(1, 1, -1)
    ).view(-1)
    sampling_probabilities = sampling_probabilities / sampling_probabilities.sum()

    sampled_bins = torch.multinomial(
      sampling_probabilities, len(env_ids), replacement=True
    )
    self.time_steps[env_ids] = (
      (sampled_bins + sample_uniform(0.0, 1.0, (len(env_ids),), device=self.device))
      / self.bin_count
      * (self.motion.time_step_total - 1)
    ).long()
