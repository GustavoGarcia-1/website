Humanoid Fighting

In Progress

Overview

This project is part of a humanoid robot fighting league run by Engine AI. The goal is to train the EngineAI T800 humanoid to perform fighting combinations in preparation for real world deployment against other robots. The pipeline converts real boxing motion capture data into joint trajectories, then uses PPO to train a policy that imitates the human performing the motion.

The system is built on MJLab, a MuJoCo based RL framework with RSL RL as the training backend. The T800 model has 25 joints tracked across 14 body segments, with self collision detection to handle violent, asymmetric motions like punches.

Mocap Data and Motion Conversion

We were given boxing motion capture data as NPY files containing base position, base quaternion, and joint angles for the T800's 25 joints. To use this data in MJLab's tracking pipeline, it needed to be converted to NPZ format with full body state information including body positions, orientations, and velocities for every tracked link.

We used the conversion script provided under the URKL official repository. The script parses the T800 URDF to build the kinematic tree, resamples frames from 30Hz to 50Hz using SLERP for quaternion interpolation, runs forward kinematics to compute world space positions and orientations for all 26 body links, then derives linear and angular velocities via finite differences.

urkl_official/scripts/npy_to_npz.py
def forward_kinematics(
    base_pos: np.ndarray, base_quat: np.ndarray,
    joint_pos: np.ndarray, incoming_joints: dict[str, JointSpec]
) -> tuple[np.ndarray, np.ndarray]:
    frames = joint_pos.shape[0]
    link_positions = np.zeros((frames, len(AVAILABLE_LINK_NAMES), 3), dtype=np.float32)
    link_quaternions = np.zeros((frames, len(AVAILABLE_LINK_NAMES), 4), dtype=np.float32)
    joint_index = {name: idx for idx, name in enumerate(T800_DFS_JOINT_NAMES)}

    for frame_idx in range(frames):
        cache: dict[str, np.ndarray] = {
            "LINK_BASE": make_transform(
                quat_to_matrix(base_quat[frame_idx]), base_pos[frame_idx]
            )
        }

        def world_transform(link_name: str) -> np.ndarray:
            if link_name in cache:
                return cache[link_name]
            spec = incoming_joints[link_name]
            parent_tf = world_transform(spec.parent_link)
            angle = float(joint_pos[frame_idx, joint_index[spec.name]]) \
                if spec.name in joint_index else 0.0
            cache[link_name] = apply_joint(parent_tf, spec, angle)
            return cache[link_name]

        for link_idx, link_name in enumerate(AVAILABLE_LINK_NAMES):
            tf = world_transform(link_name)
            link_positions[frame_idx, link_idx] = tf[:3, 3]
            link_quaternions[frame_idx, link_idx] = matrix_to_quat_wxyz(tf[:3, :3])

    return link_positions, link_quaternions

Reward Shaping

The reward functions come from MJLab's existing G1 tracking implementation, which we reused for the T800. Each reward term computes an exponential Gaussian kernel over the squared error between the reference motion and the robot's current state, providing smooth gradients everywhere even far from the target.

There are six tracking terms: anchor body position and orientation (weighted at 0.5 each), relative body position and orientation across all 14 tracked segments (weighted at 1.0 each), and linear and angular velocity matching (weighted at 1.0 each). On top of those, an action rate penalty discourages jerky motions, a joint limit penalty keeps joints in bounds, and a self collision cost penalizes the robot from intersecting its own limbs.

mjlab/src/mjlab/tasks/tracking/mdp/rewards.py
def motion_global_anchor_position_error_exp(
    env: ManagerBasedRlEnv, command_name: str, std: float
) -> torch.Tensor:
    command = cast(MotionCommand, env.command_manager.get_term(command_name))
    error = torch.sum(
        torch.square(command.anchor_pos_w - command.robot_anchor_pos_w), dim=-1
    )
    return torch.exp(-error / std**2)


def motion_relative_body_position_error_exp(
    env: ManagerBasedRlEnv, command_name: str,
    std: float, body_names: tuple[str, ...] | None = None,
) -> torch.Tensor:
    command = cast(MotionCommand, env.command_manager.get_term(command_name))
    body_indexes = _get_body_indexes(command, body_names)
    error = torch.sum(
        torch.square(
            command.body_pos_relative_w[:, body_indexes]
            - command.robot_body_pos_w[:, body_indexes]
        ), dim=-1,
    )
    return torch.exp(-error.mean(-1) / std**2)


def self_collision_cost(
    env: ManagerBasedRlEnv, sensor_name: str,
    force_threshold: float = 10.0,
) -> torch.Tensor:
    sensor: ContactSensor = env.scene[sensor_name]
    data = sensor.data
    if data.force_history is not None:
        force_mag = torch.norm(data.force_history, dim=-1)
        hit = (force_mag > force_threshold).any(dim=1)
        return hit.sum(dim=-1).float()
    return data.found.squeeze(-1)

PPO Configuration and Training

Training uses PPO via RSL RL with a 3 layer MLP (512, 256, 128) for both actor and critic using ELU activations and observation normalization. The policy outputs a Gaussian action distribution with adaptive KL based learning rate scheduling that targets a KL divergence of 0.01. The agent trains for up to 30,000 iterations with 24 environment steps per rollout, checkpointing every 500 iterations.

The T800 config was adapted from MJLab's existing G1 tracking setup. The main changes were swapping the robot model, remapping joint and body names to the T800's kinematic tree, configuring the self collision sensor for the T800's link structure, and adjusting the anchor body to LINK_WAIST_YAW.

    )

T800 Environment and Body Tracking

The environment tracks 14 body segments on the T800: the base link, left and right hips/knees/ankles, the waist, and left and right shoulders/elbows. The anchor body is set to LINK_WAIST_YAW. Self collision detection uses a contact sensor with 4 step force history to catch collisions during fast motions. Domain randomization includes base center of mass perturbation and encoder bias noise.

mjlab/src/mjlab/tasks/tracking/config/t800/env_cfgs.py
def engineai_t800_flat_tracking_env_cfg(
    play: bool = False,
) -> ManagerBasedRlEnvCfg:
    cfg = make_tracking_env_cfg()
    cfg.scene.entities = {"robot": get_t800_robot_cfg()}

    self_collision_cfg = ContactSensorCfg(
        name="self_collision",
        primary=ContactMatch(mode="subtree", pattern="LINK_BASE", entity="robot"),
        secondary=ContactMatch(mode="subtree", pattern="LINK_BASE", entity="robot"),
        fields=("found", "force"),
        reduce="none",
        num_slots=1,
        history_length=4,
    )

    motion_cmd = cfg.commands["motion"]
    motion_cmd.anchor_body_name = "LINK_WAIST_YAW"
    motion_cmd.body_names = (
        "LINK_BASE",
        "LINK_HIP_ROLL_L",     "LINK_KNEE_PITCH_L",   "LINK_ANKLE_ROLL_L",
        "LINK_HIP_ROLL_R",     "LINK_KNEE_PITCH_R",   "LINK_ANKLE_ROLL_R",
        "LINK_WAIST_YAW",
        "LINK_SHOULDER_ROLL_L", "LINK_ELBOW_PITCH_L",  "LINK_ELBOW_YAW_L",
        "LINK_SHOULDER_ROLL_R", "LINK_ELBOW_PITCH_R",  "LINK_ELBOW_YAW_R",
    )

Future Work

The current policy can imitate the punch motion but there is still a lot of work to do before competition readiness. I've done most of the work so far, I am pleased with the progress and excited for what my teammates will contribute. Here is what we are working on next:

  • Training on additional fighting motions from the URKL dataset including kicks and combos
  • Tuning reward weights to improve tracking quality on fast asymmetric motions where the arms and legs move in opposite directions
  • Curating our own datasets to get more diverse training examples and better models
  • Getting sim to real transfer working on the physical T800 hardware
  • Adding an opponent agent and training reactive fighting behaviors rather than just replaying fixed motion clips
  • Winning the competition prize of a 1 million dollar gold belt
