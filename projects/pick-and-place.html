Pick and Place Robot

Overview

This project applies deep reinforcement learning to train a simulated Panda robot arm for pick and place manipulation as part of Triton Droids. Using MuJoCo MJX and Brax for GPU accelerated physics simulation, I built on a fully parallelized training pipeline in JAX that runs 2,048 environments simultaneously, accumulating 160M+ environment steps.

The core challenge was reward shaping and designing a multi phase shaped reward function that guides the agent through reaching, grasping, transporting, and placing a cube into a bin without inducing reward hacking opportunities or local optima. I started with the Brax MJX environment structure and referenced ManiSkill's PlaceSphere reward shaping approach for the phase gated reward ladder.

Environment Design

The environment was built on MuJoCo MJX, which compiles physics simulation to XLA for GPU execution via JAX. This allowed running 2,048 parallel rollouts on a single GPU, dramatically accelerating data collection compared to CPU based simulation.

The task involves a Panda robot arm that must pick up a cube from a randomized starting position and place it into a bin. Observations include joint positions, velocities, end effector pose, cube pose, bin position, and relative distances between the gripper, cube, and bin. Actions control joint torques directly.

Phase transitions are detected by three boolean checks each step with grasp detection (gripper closed and near the cube), on bin detection (cube above the bin within XY tolerance), and success detection (cube placed inside the bin).

class PickAndPlace(PipelineEnv):
    def _init_env(self) -> None:
        self.init_q = jp.array(self.sys.mj_model.keyframe("home").qpos)
        self.nu = self.sys.nu
        self.nq = self.sys.nq
        self.nv = self.sys.nv
        self.tcp_body_id = self._safe_body_id("panda_hand")
        self.cube_body_id = self._safe_body_id("cube")
        self.bin_body_id = self._safe_body_id("bin")

    def check_grasp(self, pipeline_state: State):
        tcp_pos = self._body_pos(pipeline_state, self.tcp_body_id)
        cube_pos = self._body_pos(pipeline_state, self.cube_body_id)
        left_finger = pipeline_state.q[7]
        right_finger = pipeline_state.q[8]
        gripper_closed = (left_finger < 0.038) & (right_finger < 0.038)
        return (jp.linalg.norm(tcp_pos - cube_pos) < 0.07) & gripper_closed

    def check_on_bin(self, pipeline_state: State):
        cube_pos = self._body_pos(pipeline_state, self.cube_body_id)
        bin_pos = self._body_pos(pipeline_state, self.bin_body_id)
        xy_dist = jp.linalg.norm(cube_pos[:2] - bin_pos[:2])
        return (xy_dist < 0.05) & (cube_pos[2] > (bin_pos[2] + 0.02))

    def check_success(self, pipeline_state: State):
        cube_pos = self._body_pos(pipeline_state, self.cube_body_id)
        bin_pos = self._body_pos(pipeline_state, self.bin_body_id)
        xy_dist = jp.linalg.norm(cube_pos[:2] - bin_pos[:2])
        return (xy_dist < 0.04) & (cube_pos[2] > (bin_pos[2] + 0.02))

Reward Shaping

The reward function uses a phase gated ladder structure where each phase has a strictly higher reward range than the previous one. This prevents the agent from exploiting early phase rewards while ignoring later objectives. The phases are reach (0 to 2), grasp (4 to 5), stillness (6 to 7), and success (13).

The reach phase uses tanh based distance shaping to provide smooth gradients at all distances. Once the cube is grasped, the reward jumps to the grasp phase which adds a placement distance component. When the cube is above the bin, the stillness phase rewards the agent for opening the gripper and minimizing cube velocity. A successful placement gives a flat bonus of 13.

def _compute_reward(self, pipeline_state, info):
    tcp_pos = self._body_pos(pipeline_state, self.tcp_body_id)
    cube_pos = self._body_pos(pipeline_state, self.cube_body_id)
    bin_pos = self._body_pos(pipeline_state, self.bin_body_id)

    left_finger = pipeline_state.q[7]
    right_finger = pipeline_state.q[8]
    cube_lin_vel = pipeline_state.qd[9:12]
    cube_ang_vel = pipeline_state.qd[12:15]

    is_grasped_bool = info["is_grasped"] > 0.5
    is_on_bin_bool = info["is_on_bin"] > 0.5
    success_bool = info["success"] > 0.5

    dist_tcp_cube = jp.linalg.norm(tcp_pos - cube_pos)
    reach_reward = 2.0 * (1.0 - jp.tanh(5.0 * dist_tcp_cube))

    dist_cube_bin = jp.linalg.norm(cube_pos - bin_pos)
    place_reward = 1.0 - jp.tanh(5.0 * dist_cube_bin)
    grasp_phase_reward = 4.0 + place_reward

    gripper_openness = (left_finger + right_finger) / 0.08
    ungrasp_reward = gripper_openness

    cube_speed = jp.linalg.norm(cube_lin_vel) + jp.linalg.norm(cube_ang_vel)
    stillness_reward = 1.0 - jp.tanh(cube_speed)

    stillness_phase_reward = 6.0 + (ungrasp_reward + stillness_reward) / 2.0
    success_reward = jp.array(13.0)

    reward = jp.where(
        success_bool, success_reward,
        jp.where(
            is_on_bin_bool, stillness_phase_reward,
            jp.where(is_grasped_bool, grasp_phase_reward, reach_reward)))

    return reward

Training Configuration

The agent was trained using PPO (Proximal Policy Optimization) via Brax's training pipeline for 40M timesteps across 2,048 parallel environments. The policy and value networks are both 256x256x256 MLPs with separate observation keys for policy and privileged state.

@dataclass
class PickAndPlaceConfig:

    @dataclass
    class PPOAgentConfig:
        num_timesteps: int = 40_000_000
        num_evals: int = 100

        unroll_length: int = 50
        num_minibatches: int = 32
        num_updates_per_batch: int = 4
        discounting: float = 0.99
        learning_rate: float = 3e-4
        entropy_cost: float = 0.02
        clipping_epsilon: float = 0.2
        num_envs: int = 2048
        batch_size: int = 1024

    class PPONetworksConfig:
        policy_hidden_layer_sizes: Tuple[int, ...] = (256, 256, 256)
        value_hidden_layer_sizes: Tuple[int, ...] = (256, 256, 256)
        policy_obs_key: str = "state"
        value_obs_key: str = "privileged_state"

Results and Takeaways

Training reached episode rewards of around 83 after 160M+ environment steps. However, the agent did not reliably achieve the full success reward, as you can see in the video it learned the general trajectory but failed to grasp the cube.

The reward ladder structure was broken in the early phases and did not provide a proper gate for moving onto the next phase. Shaping the rewards from the initial system I tried proved challenging, the gated structure I switched to looks more promising than before.

Training was tracked via Weights and Biases, with checkpoints saved throughout. The parallelized JAX pipeline made it feasible to run the full 40M timestep training in a reasonable amount of time on a single GPU.

While the agent struggled, I gained exposure to simulators and reinforcement learning techniques. I'm excited to utilize these skills in future projects.
