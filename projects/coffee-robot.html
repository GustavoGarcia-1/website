Coffee Making Robot

Overview

This project tackles multi-step robotic manipulation for an everyday task: making coffee. Using the so101 robot arms and the LeRobot framework, I collected a teleoperation dataset covering the full coffee-making sequence — from grasping the cup, to positioning it under the machine, to operating the brew button. The goal was to train a vision-language-action policy that could reproduce this long-horizon behavior from learned demonstrations.

I used Physical Intelligence's π0.5 flow matching architecture as the backbone, fine-tuning it on the collected dataset. A key focus was on action normalization strategies, comparing quantile normalization against mean-std normalization to understand their effects on prediction fidelity and trajectory smoothness.

Data Collection Pipeline

Demonstration data was collected by teleoperating the so101 arms through the full coffee-making sequence. Each episode was captured from multiple synchronized camera angles to provide rich visual context for the policy. I implemented temporal task segmentation to break long-horizon sequences into distinct phases (reach, grasp, pour, place), which helped the model learn phase transitions more reliably.

The dataset was uploaded to HuggingFace for version control and reproducibility.

# Example: LeRobot dataset recording configuration
from lerobot.common.datasets import LeRobotDataset

    repo_id="gustavo/coffee-making-so101",
    fps=30,
    video=True,
    image_keys=["observation.images.top", "observation.images.wrist"],

Model Architecture & Training

The π0.5 flow matching policy was fine-tuned on the collected dataset using the LeRobot training pipeline. Flow matching generates smooth, continuous action trajectories by learning a vector field that maps noise to demonstrations — making it well-suited for precise manipulation tasks.

I experimented with two normalization strategies for the action space: quantile normalization (which maps actions to a uniform distribution) and mean-std normalization (which centers and scales by standard deviation). Quantile normalization proved more robust for tasks with multi-modal action distributions, while mean-std worked well for smoother, unimodal sequences.

# Example: Training configuration for pi0.5
training:
  policy: "pi0.5"
  batch_size: 32
  lr: 1e-4
  num_epochs: 200
  normalization: "quantile"   # vs "mean_std"
  action_chunk_size: 50

Results & Takeaways

The trained policy was able to execute the full coffee-making sequence with reasonable success rate from the learned demonstrations. Key findings included the importance of camera placement for consistent visual features, and the significant impact that normalization choice has on action prediction quality — especially for tasks involving multiple distinct motion phases.

Future work includes expanding the dataset with more diverse episodes and testing policy generalization across different cup placements and coffee machine configurations.
